semantic 태그

의미가 있는

  • html4버전에서는 div태그의 id를 통해 내용을 구분
  • 스크린리더는 영역별로 접근 불가
  • html5버전에서 태그의 이름을 영역을 유추할 수 있도록 만듦
  • 스크린리더가 영역별로 접근
  • 웹 접근성 향상을 위한 태그

사용 태그

    <header>
        헤더 부분 : 맨 위 상단에 위치하고, 회사 로고나 기타 헤더에 넣을 것들을 배치
    </header>
    <nav>
        nav : 다른 페이지로 이동할 수 있는 페이지의 네비게이션 정보. 위치가 가장 자유롭다
    </nav>
    <section>
        section : 연관된 컨텐츠 그룹핑
    </section>
    <article>
        실제 내용을 
        - section으로 여러개의 article이 그룹핑 가능
        - aricle본문이 section으로 나뉠 수 있음
    </article>
    <side>
        aside : 본문 외의 부가정보
        - 날씨, 광고, 등
    </side>
    <footer>
        footer : 사이트 하단 부분
        - 저작권 정보, 회사 정보
    </footer>
